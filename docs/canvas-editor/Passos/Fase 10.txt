FASE 10 — DOCUMENT ENGINE & MULTI-PAGE PROFISSIONAL
Objetivo

Transformar o Canvas atual de "editor de slide único" para "editor de documento/carrossel completo".

O usuário deve conseguir:

criar slides
duplicar slides
deletar slides
reordenar slides
navegar rapidamente
visualizar miniaturas
trabalhar com 20+ páginas sem perda de performance
Regra Arquitetural nº 1

Não criar lógica de páginas dentro do Canvas.

Criar camada própria:

src/services/document/

Separar:

createPage.ts
duplicatePage.ts
deletePage.ts
reorderPages.ts
generateThumbnail.ts
Regra Arquitetural nº 2

O Canvas continua renderizando SOMENTE:

activePage

Nunca:

pages[]

inteiro.

Regra Arquitetural nº 3

Miniaturas não podem renderizar Konva completo.

Seria um desastre de performance.

Implementar:

thumbnail cache

Estrutura:

PageThumbnail {
  pageId: string
  preview: string
  updatedAt: number
}

As miniaturas devem ser:

PNG
baixa resolução
cacheadas

Geradas apenas:

criação
duplicação
edição concluída

Nunca durante drag.

Criar slide funcionar.

Reordenar funcionar.
