# Missão: Remover Canvas Atual e Preparar Estrutura para OpenPolotno

## Contexto

O Carrossel Studio passou por uma auditoria completa e a decisão arquitetural já foi tomada:

**O OpenPolotno será o editor oficial de canvas do produto.**

Existe atualmente um editor/canvas próprio desenvolvido anteriormente durante tentativas de replicar funcionalidades similares ao Polotno. Esse editor não será aproveitado.

O objetivo desta tarefa NÃO é integrar o OpenPolotno ainda.

O objetivo é limpar a arquitetura atual para receber a nova engine sem herdar código legado desnecessário.

---

# Nova Estrutura do Produto

A navegação principal deve ficar assim:

* Início
* Studio
* Canvas
* Mídia

## Início

Página de boas-vindas premium.

Responsável apenas pela apresentação do produto.

Nenhuma funcionalidade de edição.

---

## Studio

Biblioteca principal de templates.

Responsável por exibir os 300+ designs disponíveis.

Nenhuma funcionalidade de edição direta.

Responsável apenas por seleção e gerenciamento da biblioteca.

---

## Canvas

Novo local oficial do editor.

O OpenPolotno será integrado aqui futuramente.

---

## Mídia

Local de adição de mídia.

Essa página será organizada e pensada posteriormente, não se preocupe com ela agora.

---

# Objetivo da Tarefa

Realizar uma remoção completa do canvas/editor atual.

Não criar compatibilidade.

Não reutilizar componentes antigos.

Não preservar lógica de edição.

Não criar wrappers.

Não criar adaptadores.

O editor atual deve ser removido de forma definitiva.

---

# Etapa 1 — Auditoria do Editor Atual

Identificar todos os arquivos relacionados ao editor existente.

Localizar:

* páginas
* componentes
* hooks
* stores
* contextos
* providers
* utilitários
* serviços
* rotas

Mapear tudo que pertence ao canvas legado.

Gerar uma lista completa antes da remoção.

---

# Etapa 2 — Remoção do Canvas Legado

Remover completamente:

* componentes do editor
* canvas customizado
* ferramentas de edição
* sistema de arrastar
* sistema de redimensionar
* sistema de camadas
* sistema de exportação
* stores exclusivas do editor
* hooks exclusivos do editor
* dependências que existam apenas para sustentar o editor atual

Objetivo:

Nenhum código morto.

Nenhum código comentado.

Nenhum código legado sobrando.

---

# Etapa 3 — Renomeação da Navegação

Substituir qualquer referência a:

Auto-post

por:

Canvas

Atualizar:

* menu lateral
* breadcrumbs
* rotas
* títulos
* metadados
* navegação interna

---

# Etapa 4 — Criar Página Placeholder

Criar uma página temporária para o Canvas.

Exemplo:

Título:
Canvas

Subtítulo:
Editor profissional em preparação.

Descrição:
Esta área receberá a integração oficial do OpenPolotno.

A página deve seguir o padrão visual premium do produto.

Não utilizar componentes do editor antigo.

---

# Etapa 5 — Validar Arquitetura

Após a remoção:

O projeto deve compilar sem erros.

Não deve existir nenhuma dependência do editor antigo.

Nenhuma rota deve apontar para componentes removidos.

Nenhum import quebrado deve permanecer.

---

# Resultado Esperado

Estrutura final:

Início
→ Página premium

Studio
→ Biblioteca de templates

Canvas
→ Página placeholder pronta para receber OpenPolotno

Mídia
→ Página à ser preparada posteriormente

Sem editor legado.

Sem código morto.

Sem dependências desnecessárias.

Sem compatibilidade com a implementação anterior.

A arquitetura deve ficar limpa e preparada para a próxima fase:

Integração do OpenPolotno como engine oficial do Carrossel Studio.
